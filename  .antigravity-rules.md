# Diretrizes de Desenvolvimento (Antigravity Rules)

Este arquivo contém as instruções permanentes que o assistente Antigravity deve seguir para este projeto. **Sempre leia este arquivo no início de cada interação.**

## 1. Idiomas e Comunicação
- **Mensagens de Commit:** Devem ser SEMPRE em **Inglês**.
- **Planos de Implementação (Artifacts):** Devem ser SEMPRE em **Português**.
- **Explicações:** Sempre explique a lógica e o que será feito ANTES de executar qualquer alteração.
- **Walkthroughs:** Devem ser em **Português**.

## 2. Fluxo de Trabalho (Workflow)
O assistente deve seguir rigorosamente esta ordem para solicitações complexas:
1. **Pesquisa:** Analisar os arquivos e o contexto.
2. **Plano de Implementação:** Criar o artifact `implementation_plan.md` em Português e solicitar aprovação.
3. **Execução:** Após aprovação, criar o `task.md` e realizar as alterações.
4. **Resumo:** Criar o `walkthrough.md` e fornecer o nome do commit em Inglês ao final.

## 3. Padrões Estéticos (UI/UX)
- **Border Radius:** O padrão global do projeto é `0.5rem` (`rounded-lg` para cards, `rounded-md` para botões/inputs). Evite arredondamentos excessivos ("estilo amador").
- **Estética:** Design moderno, minimalista e premium, inspirado em interfaces como **ChatGPT (OpenAI)**, **Vercel** e **Linear**.
- **Menu de Perfil:** Seguir a estética "flat" sem fundos coloridos nos ícones, com cantos `rounded-xl` e sem animações de abertura.

## 4. Controle de Versão
- Nunca commite este arquivo (`.antigravity-rules.md`) - ele deve permanecer apenas local (já adicionado ao `.gitignore`).
- Sempre peça confirmação do nome do commit ao final das tarefas.
